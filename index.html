SPA and SEO
SPA advantages:
  • High speed
  • universalism and availability
  • flexibility and responsiveness
  • no reload and re-render
Popular frameworks:
  • React
  • Vue
  • Angular
SPA problems:
  • A lot of time first renders
  • SEO is difficult
  • The search robot sees the site as empty
The proplem: render blank page on load
  • Static Site Generation (SSG)
  • Server-Side Rendering (SSR)
Static Site Generation (SSG)

Pre-render the site before uploading it to the server.

  • Great for simple web pages without interacting with backend and API
Server-Side Rendering (SSR):

Render content at runtime on the server

  • With this approach, we will be able to make backend API requests and return HTML along with the required content.
Server-Side Rendering (SSR)
  • Server (start app as user in browser)
  • Server make requests, get resourses, create HTML and fill the state
  • Server send the HTML and the resourses
  • Client gets the resourses, synchronizes state and work as SPA.
Problems SSR:
  • The App is devided into Client and Server - support is difficult
  • Async requests on Server
  • Synchronization
Bottom line
  • SEO-friendly site and SPA is possible
  • Estimate your App and choose the way
